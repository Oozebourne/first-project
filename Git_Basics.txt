git init - инициализация репозитория
git status - статус репозитория
git add - добавить файл в репозиторий
git add --all либо git add . - добавить все файлы в папке в репозиторий
git commit -m 'Message' - запомнить изменения в файлах репозитория, добавить в историю изменений с сообщением Message.
git remote add name SSH_URL - привязать к удалённому репозиторию SSH_URL локальный репозиторий name.
git remote -v - подробная информация о привязанном репозитории.
git push - отправить изменения на привязанный удалённый репозиторий.
git log - история коммитов.
git log --oneline - короткий лог коммитов.

README.md - стандартное название файла для краткого описания проекта. Размечено языком Markdown.
Хеширование - получение уникального отпечатка (fingerprint) с помощью хеш-функции. Хеш-функция всегда возвращает одно и то же значение при обработке одного и того же набора входных данных, где бы она ни использовалась.
Git использует алгоритм хеширования SHA-1 (Secure Hash Algorithm).
HEAD - служебный файл в папке  .git, указывающий на последний (самый новый) коммит. Внутри - ссылка на служебный файл, в котором хранится хеш последнего коммита. Также HEAD может использоваться как ссылка на хеш последнего коммита.

Статусы файлов в Git:
 - untracked
 - staged
 - tracked
 - modified
git status отображает ветку, файлы в папке репозитория и их статус (кроме tracked).
